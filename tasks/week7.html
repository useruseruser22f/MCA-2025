Week 7 – Curating and Displaying Metadata

In Week 7 I revisited the metadata for my dataset theme (Cantabile in B-flat major, B.84 by Frédéric Chopin) and focused on two curatorial tasks: (1) enriching the MEI header with genre and licensing information, and (2) learning how to display that metadata on a web page alongside the engraved score using Verovio and CETEIcean.

Task 1 – Updating the MEI Metadata

I started from the MEI file I created in Week 5 and saved a new version for this lab (chopin-cantabile-meta.mei in my data/week7/ folder). I then edited the <meiHead> to include two additional types of metadata: a genre classification and an explicit licence.

Genre

I added a genre description inside the <notesStmt> as an extra annotation: 

<notesStmt>
  <annot>
    Encoded for MCA-2025 metadata work. Schema includes title,
    composer, publisher, date, genre and catalogue number.
  </annot>
  <annot type="genre">Character piece (cantabile piano work)</annot>
</notesStmt>

I chose the label “character piece” because this work is a short, lyrical piano solo and is often grouped with Chopin’s other smaller character pieces (like nocturnes and preludes). Recording this genre in the header means future users can filter my dataset not only by composer and key, but also by the type of piece.

Licensing

I also added an <availability> element indicating how the encoding may be reused: 

<availability>
  <licence target="https://creativecommons.org/licenses/by-sa/4.0/">
    This MEI encoding is released under a Creative Commons
    Attribution-ShareAlike 4.0 International (CC BY-SA 4.0) licence.
  </licence>
</availability>

I chose a CC BY-SA licence because the source score is in the public domain, but I still want people who reuse my encoding to give credit and to share any derivative encodings under the same open licence. This is a realistic choice for teaching and research datasets, where reuse and adaptation are encouraged but attribution still matters.

After adding these elements I revalidated the MEI in Verovio to ensure there were no schema errors.

Task 2 – Rendering Metadata and Score in HTML

For the second task I connected this enriched MEI file to the metaRAW.html page in my repository. In that file I replaced the default example path (data/example.mei) with the path to my new MEI file: 

var meiFile = "data/week7/chopin-cantabile-meta.mei";

When I opened [myusername].github.io/MCA-2025/metaRAW.html in the browser, the MEI header was displayed as a long block of unformatted text above the Verovio score. This shows that the metadata is present and readable, but not yet very user friendly.

How I Would Improve the Formatting

To make the metadata more usable on the web, I would:

  • Group the header into sections such as “Work information”, “Publication details” and “Licensing”.
  • Display important fields (title, composer, genre, licence) as a small table or definition list instead of raw text.
  • Highlight the licence with a short label (e.g. “CC BY-SA 4.0”) and make the link clickable.
  • Use CSS to adjust spacing, font size and bold labels so that the metadata is easy to skim.

The optional part of the lab suggests two approaches: using CSS with CETEIcean (e.g. based on metaCSS1.html) or using Javascript (based on metaJS.html). If I extend this work later, I would probably use the CSS option, because it keeps the logic in the MEI file and styles the existing tags rather than manually rebuilding the metadata with Javascript.

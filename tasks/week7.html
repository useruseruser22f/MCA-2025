Week 7 – Curating and Displaying Metadata

In this lab I revisited the metadata for my dataset theme, Frédéric Chopin’s Cantabile in B-flat major, B.84, and focused on two things: (1) enriching the MEI header with genre and licensing information, and (2) learning how to display that metadata on a web page alongside the engraved score using Verovio and CETEIcean.

Task 1 – Updating the MEI Metadata

I began with my Week 5 MEI file and exported a new version for this lab, which I saved as chopin-cantabile-meta.mei in my data/week7/ folder. In this new file I edited the <meiHead> to add a genre classification and explicit licensing information.

Genre

To record the genre, I added an extra annotation inside <notesStmt>: 

<notesStmt>
  <annot>
    Encoded for MCA-2025 metadata work. Schema includes title,
    composer, publisher, date, genre and catalogue number.
  </annot>
  <annot type="genre">Character piece (cantabile piano work)</annot>
</notesStmt>

I chose the label “character piece (cantabile piano work)” because the piece is a short, lyrical solo and is often grouped with Chopin’s other intimate character pieces. Recording this in the MEI header means that my catalogue could later be filtered or searched by genre as well as by composer, key or date.

Licensing

I also added licensing information inside <pubStmt> using an <availability> element: 

<pubStmt>
  <publisher>IMSLP- International Music Score Library Project</publisher>
  <pubPlace>Paris</pubPlace>
  <date isodate="1834">1834</date>

  <availability>
    Licensed under Creative Commons Attribution-ShareAlike 4.0 International
    (CC BY-SA 4.0). See: https://creativecommons.org/licenses/by-sa/4.0/
  </availability>
</pubStmt>

I chose a Creative Commons Attribution-ShareAlike 4.0 licence because the original score is in the public domain, but I still want to encourage proper attribution of the encoding and allow others to reuse and adapt it as long as they share any derivatives under the same open licence. This is a realistic choice for an educational and research-oriented dataset.

After these changes I checked the header in the Verovio XML view and adjusted the structure (for example, placing <availability> inside <pubStmt>) until no red validation errors remained.

Task 2 – Rendering Metadata and Score on the Web

For the second task I connected this enriched MEI file to the metaRAW.html page provided in the repository. In that file I replaced the example path with the location of my new MEI file, e.g.: 

var meiFile = "data/week7/chopin-meta.mei";

When I opened useruseruser22f.github.io/MCA-2025/metaRAW.html in the browser, CETEIcean rendered the MEI header as a block of unformatted text above the Verovio score. This confirmed that my metadata was correctly encoded and being read, but it also showed that the default display is not very user friendly.

How I Would Format the Metadata

To make the metadata more readable for visitors to the site, I would:

  • Group related fields into sections such as “Work details” (title, composer, genre), “Publication” (publisher, place, date), and “Licensing”.
  • Use headings and a simple table or list to separate labels (e.g. “Composer:”) from their values.
  • Highlight the licence as a short badge (for example “CC BY-SA 4.0”) and make the URL to the Creative Commons page clickable.
  • Apply CSS (for example in CETEIcean.css) to adjust font sizes, margins and spacing so the metadata block feels like a clean information panel rather than a chunk of text.

The optional part of the lab suggests using either CSS (via templates like metaCSS1.html) or Javascript (via metaJS.html) to format the metadata automatically. If I extend this work later, I will probably choose the CSS-based approach, because it lets me keep the structure in MEI and simply change how those elements are displayed, rather than rebuilding the metadata manually in Javascript.
